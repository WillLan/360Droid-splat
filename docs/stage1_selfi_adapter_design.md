# Stage 1 Spherical Selfi Adapter Design

## Goal

Stage 1 trains a standalone spherical feature adapter for ERP images. It follows
Selfi's geometric feature alignment idea, but replaces planar pixel reprojection
supervision with unit-sphere bearing supervision so the loss is valid across the
ERP seam and at high latitude.

The Stage 1 modules are not wired into the SLAM frontend, mapper, backend, or
`FrontendOutput`. They are training-time components only.

## Selfi Reference

Selfi adds a DPT-style adapter on top of frozen VGGT image tokens, uses four
intermediate feature stages, predicts dense feature maps, and supervises feature
matching from pseudo correspondences generated by VGGT depth and camera poses.

For this project the same structure is kept, while the correspondence target is
represented as a target-view unit ray:

```text
source ERP pixel + source ray depth + source T_c2w
  -> world point
  -> target camera point
  -> target target_ray on S^2
```

The matching loss predicts a target bearing by softmax-weighted feature
similarity and minimizes the great-circle distance between the predicted target
bearing and the pseudo target bearing.

Reference links:

- https://arxiv.org/abs/2512.08930
- https://denghilbert.github.io/selfi/

## Frozen PanoVGGT Wrapper

Implemented in `models/panovggt_feature_wrapper.py`.

Contract:

- The wrapped PanoVGGT-like model is put in `eval()` mode.
- All wrapped model parameters have `requires_grad=False`.
- Forward runs under `torch.no_grad()` by default.
- Exactly four hook names are configured.
- Hook outputs are normalized to `B x V x C x H x W`.
- Token outputs can be reshaped from square token grids, explicit `grid_hw`, or
  already-spatial feature maps.
- Initial depth, camera-to-world pose, and optional world points are preserved if
  the wrapped model returns them.

The wrapper intentionally does not hard-code real PanoVGGT layer names. Real hook
names are checkpoint/model-version dependent and must be verified with:

```bash
python tools/dump_panovggt_feature_shapes.py --config configs/stage1_spherical_selfi_adapter.yaml
```

For local smoke tests without a checkpoint, use:

```bash
python tools/dump_panovggt_feature_shapes.py --config configs/stage1_spherical_selfi_adapter.yaml --synthetic --device cpu
```

## Spherical DPT Adapter

Implemented in `models/spherical_selfi_dpt_adapter.py`.

Contract:

- Input is a list of four feature tensors, each `B x V x C_i x H_i x W_i`.
- Each stage is projected to a shared hidden dimension.
- Fusion is top-down and DPT-style.
- Convolutions use ERP-aware padding: circular horizontally, replicate
  vertically.
- Output shape defaults to `B x V x 24 x 504 x 1008`.
- Output resolution is configurable and not inferred as a fixed global constant.
- Output descriptors are L2-normalized along the channel dimension.

The adapter output resolution is set by config:

```yaml
adapter:
  out_dim: 24
  output_height: 504
  output_width: 1008
```

## Spherical Alignment Loss

Implemented in `losses/spherical_selfi_alignment_loss.py`.

Main residual:

```text
theta = acos(clamp(dot(normalize(pred_ray), normalize(target_ray)), -1, 1))
```

This is the unit-sphere geodesic / great-circle angular distance. It is the main
loss term:

```text
loss = weighted_mean(theta) + erp_aux_weight * erp_aux
```

`erp_aux` is a small seam-aware pixel auxiliary term for diagnostics and
stabilization. It is not the main reprojection residual.

Supported matching modes:

- `global_lowres`: predicts a target bearing from a downsampled target feature
  grid.
- `local_fullres`: predicts a target bearing inside a local ERP window around
  the pseudo target pixel.

## Pose and Depth Convention

Stage 1 uses camera-to-world poses `T_c2w`.

Depth is Euclidean ray depth along the ERP unit ray:

```text
X_cam = range * source_ray
```

The code does not silently interpret z-depth as ray depth. If a dataset stores
z-depth, the caller must convert it explicitly before generating pseudo
correspondences.

## Scope Boundary

Implemented:

- ERP spherical geometry.
- Source-target spherical pseudo correspondence generation.
- Frozen PanoVGGT feature wrapper.
- Spherical DPT adapter.
- Spherical Selfi alignment loss.
- Manifest dataset.
- Training entrypoint.
- Feature shape dump, overlap check, and match visualization tools.

Not implemented in Stage 1:

- SLAM frontend integration.
- Dense BA integration.
- Mapping/backend changes.
- `FrontendOutput` API changes.
- Real experiment launch on remote server.
